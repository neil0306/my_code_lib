# redis 基础知识
redis 是一款基于内存的数据库，主要目的是处理高并发的场景。
- redis 在执行增删改查命令的时候是`单线程`的
- 由于 redis 是基于内存数据库，因此在没有任何持久化措施的情况下，断电就会丢失数据，为了防止断电数据丢失，redis 会通过`RDB`和`AOF`这两种文件进行持久化。

## RDB 文件
RDB 文件可以理解为`数据快照`, 它存储的是 redis 当前的数据快照，这些数据以`二进制文件`的方式存储，它是`全量`数据快照，也就是说 redis 会把所有的数据都塞进 RDB 这个文件中，因此它通常较大。
- 在保存数据时，redis 提供了`save`和`bgsave`
  - 上面说了，reids 执行增删改查命令时是单线程的，也就是主线程，save 这个命令就是运行在主线程的命令数据保存命令，此时，如果 redis 数据量大，则使用 save 命令时特别容易造成阻塞！因此**一般不使用 save 去保存 RDB 数据**。

  - bgsave 指的是 background save，它会**开一个子线程在后台进行数据保存**。
    - 但与此同时还有一个问题，每次数据发生变化，而 RDB 就是要存储全量数据的，因此这就会引申出保存数据的频率问题，通常会设置频率为 5 分钟。
    - 在子线程保存数据的时候，主线程怎么写新数据呢？这里 redis 使用的是`Copy On Write(写入即复制)`, 也就是搞一份数据拷贝，主线程每次写的是数据拷贝，而负责保存的子线程会将原始数据持续写入 RDB 文件中。
        > 这是一种常见的并发编程的线程安全里的`无锁`思想。


## AOF 文件
RDB 文件是全量数据的二进制文件，这会消耗大量磁盘空间造成磁盘 IO 压力，因此 redis 又提供了另一种数据持久化方式，那就是`将增删改查这些修改数据库的命令保存下来`, 这样 redis 重启的时候把这些命令再快速执行一遍就可以实现持久化了。
- 注意虽然这里写的是执行的命令日志，但其实还是`先写到一个缓冲区, 然后再写入磁盘`的。此时的执
行顺序是：
  1. redis 主线程执行增删改查命令 (优先执行命令而不是写缓存，是因为 redis 要优先保证速度); 
  2. redis 主线程把命令写入缓冲区; 
  3. redis 主线程或操作系统将缓冲区的数据写入磁盘。
      - 写盘时间通过 redis 的 config 文件去调整，分为 aways(每次主线程执行完增删改查命令，命令写入了缓冲区后，主线程就去主动写盘); No(redis 主线程执行完增删改查命令后，命令写入缓冲区，由操作系统自己决定写盘时机); EverySecond(执行命令，写入缓冲，然后 redis 主线程每秒写一次盘)

# 在 Python 程序中检查 redis 中的某个 key 数据的过期时间
```
# TTL 返回值含义：
import redis

redis_conn = redis.Redis(unix_socket_path=None, db=0, charset="utf-8")  # 连接到redis数据库

ttl = redis_conn.ttl('key_name')

if ttl == -1:
    print("键永不过期")
elif ttl == -2:
    print("键不存在")
else:
    print(f"键将在 {ttl} 秒后过期")
```
